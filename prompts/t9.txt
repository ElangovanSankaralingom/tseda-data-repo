## The Fix

### Step 1: Auto-categorize

Run this diagnostic to find the most duplicated type names:
```bash
grep -rn "^type \|^export type \|^interface \|^export interface " components/ hooks/ --include="*.tsx" --include="*.ts" | \
  sed 's/.*\(type\|interface\) \([A-Za-z]*\).*/\2/' | sort | uniq -c | sort -rn | head -20
```

Types appearing more than once are definitely candidates for extraction.

### Step 2: Create/extend canonical type files

Ensure these exist in `lib/types/`:
- `entry.ts` — CanonicalEntry, EntryApiResponse, EntryStatus, FieldStage (from P1.1)
- `admin.ts` — UserProfile, UserStats, AdminNotification, AuditFinding, IntegrityReport
- `dashboard.ts` — DashboardSummary, CategoryProgress, StreakSnapshot
- `category.ts` — CategoryKey, CategoryMetadata, CategorySchema
- `export.ts` — ExportConfig, ExportTemplate, ExportFilter
- `ui.ts` — Shared UI prop types if any (StatusColor, BadgeVariant, etc.)

### Step 3: Move types in batches

Process by file — open each component that has inline types:
1. If the type is used only in that one file AND is purely UI state → KEEP
2. If the type represents a domain concept (Entry, User, Status, etc.) → MOVE to lib/types/
3. If the type is a duplicate of one in lib/types/ → DELETE and import

### Step 4: Target count
After this prompt, the count should be:
```bash
grep -rn "^type \|^interface " components/ --include="*.tsx" | wc -l
# Target: under 40
```
