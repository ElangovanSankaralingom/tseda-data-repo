### 2. Engine Tests (COMPREHENSIVE)
`tests/entries/engine.test.ts` — minimum 30 test cases:

**Create:**
- Creates with correct initial fields
- Generates unique ID
- Sets DRAFT status
- Rejects missing required fields

**Update:**
- Updates allowed fields
- Sets updatedAt
- Rejects update on finalized entry
- Rejects update on archived entry
- Sanitizes string fields
- Rejects protected field overwrite (confirmationStatus, etc.)

**Generate (DRAFT→GENERATED):**
- Sets correct status, editWindowExpiresAt, committedAtISO
- Sets streakEligible=true when endDate is future
- Sets streakEligible=false when endDate is past
- Sets pdfGenerated, pdfGeneratedAt
- Rejects generate on non-DRAFT entry

**Finalise:**
- Succeeds for GENERATED with valid PDF
- Sets permanentlyLocked on second finalization
- Rejects without PDF
- Rejects if PDF stale
- Rejects on DRAFT entry

**Request Edit:**
- Transitions to EDIT_REQUESTED
- Increments requestCount
- Sets streakPermanentlyRemoved on Win entry
- Rejects at limit (3/month)
- Rejects on non-finalized entry

**Request Delete:**
- Transitions to DELETE_REQUESTED
- Cancels pending edit request
- Rejects on non-finalized entry

**Delete:**
- Deletes DRAFT
- Deletes editable GENERATED
- Rejects finalized
- Removes from store completely

**Admin: Grant, Reject, Archive, Restore**
- Each with positive and negative cases
