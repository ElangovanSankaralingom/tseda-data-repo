## The Implementation — EVERY ITEM MANDATORY

### 1. GitHub Actions CI
Create `.github/workflows/ci.yml`:
```yaml
name: CI
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  quality:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 24, cache: 'npm' }
      - run: npm ci
      - run: npm run lint
      - name: Type check
        run: npx tsc --noEmit
      - run: npm run build
      - name: Tests with coverage
        run: npm run test:ci
        env: { NODE_ENV: test }
      - name: Security audit
        run: npm audit --audit-level=high

  docker:
    runs-on: ubuntu-latest
    needs: quality
    steps:
      - uses: actions/checkout@v4
      - run: docker build -t tseda:ci .
```

### 2. Release Workflow
Create `.github/workflows/release.yml`:
```yaml
name: Release
on:
  push:
    tags: ['v*']
jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 24, cache: 'npm' }
      - run: npm ci
      - run: npm run build
      - run: npm test
      - uses: softprops/action-gh-release@v2
        with:
          generate_release_notes: true
```

### 3. Branch Protection (document)
Add to CONTRIBUTING.md:
```
## Branch Protection Rules (set in GitHub Settings)
- Require pull request reviews before merging
- Require status checks to pass (CI quality job)
- Require branches to be up to date before merging
- Do not allow force pushes to main
```

### 5. Docker
Create `Dockerfile`:
Multi-stage build: deps → builder → runner.
Use standalone output. Create .data dir. Run as non-root user.
Expose port 3000. Health check configured.

Create `docker-compose.yml`:
App service with volumes for .data and uploads. Health check. Restart policy.

Create `.dockerignore`:
Exclude node_modules, .next, .git, .data, tests, prompts, *.md.

Add `output: 'standalone'` to next.config.ts.
