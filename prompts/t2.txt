## The Fix

### Step 1: Diff adapters against the base

```bash
# Find functions/sections unique to case-studies
grep -n "function\|const.*=.*=>\|return" components/data-entry/adapters/case-studies.tsx | head -30

# Compare with base
grep -n "function\|const.*=.*=>" components/data-entry/adapters/BaseEntryAdapter.tsx | head -30
```

### Step 2: Identify category-specific vs duplicated code

For each large adapter, categorize every section:
- **Duplicated**: Same logic as BaseEntryAdapter → DELETE, use base
- **Category-specific field rendering**: Different fields → Should be driven by schema, not hardcoded
- **Truly unique UI**: Special layout, custom widgets → Keep, but extract into a small category-specific component

### Step 3: Absorb missing features into BaseEntryAdapter

If the base is missing features that case-studies and guest-lectures need:
- Add the feature to BaseEntryAdapter with a prop or schema-driven flag
- Example: if case-studies has a special "methodology" section with custom layout, add a `renderCustomSection` prop to the base

### Step 4: Slim down each adapter

**Target: each adapter under 50 lines** (thin wrapper around BaseEntryAdapter)

```typescript
// components/data-entry/adapters/case-studies.tsx
import BaseEntryAdapter from './BaseEntryAdapter';

export default function CaseStudiesAdapter(props) {
  return <BaseEntryAdapter category="case-studies" {...props} />;
}
```

If truly unique rendering exists:
```typescript
export default function CaseStudiesAdapter(props) {
  return (
    <BaseEntryAdapter
      category="case-studies"
      {...props}
      customSections={{
        methodology: (value, onChange) => <MethodologyEditor value={value} onChange={onChange} />
      }}
    />
  );
}
```
