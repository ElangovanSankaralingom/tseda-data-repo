## The Implementation

For EACH missing function identified in the gap report, implement it in engine.ts following this pattern:

```typescript
export async function operationName(
  email: string,
  category: CategoryKey,
  entryId: string,
  payload: Record<string, unknown>,
  options?: { nowISO?: string }
): Promise<CanonicalEntry> {
  // 1. Acquire user lock
  return withUserLock(email, async () => {
    // 2. Read the category store
    const store = await readCategoryStore(email, category);

    // 3. Find the entry
    const entry = store.byId[entryId];
    if (!entry) throw new NotFoundError(`Entry ${entryId} not found`);

    // 4. Validate the operation is allowed (status checks via workflow.ts)
    // 5. Apply the mutation (transition via workflow.ts if status changes)
    // 6. Set lifecycle fields (updatedAt, etc.)
    // 7. Write the store atomically
    // 8. Update the user index
    // 9. Append to WAL
    // 10. Return the updated entry
  });
}
```

### Key functions likely needed (verify against gap report):

**If missing: `createEntry(email, category, initialData)`**
- Generate UUID
- Set: id, category, ownerEmail, confirmationStatus: 'DRAFT', createdAt, updatedAt, schemaVersion
- Merge initialData (category-specific fields)
- Add to store.byId and store.order
- Write store, update index, append WAL

**If missing: `updateEntry(email, category, entryId, updates)`**
- Read entry, check isEntryEditable
- Merge updates into entry (only allow data fields, not lifecycle fields)
- Set updatedAt
- Write store, update index, append WAL

**If missing: `generateEntry(email, category, entryId)`**
- Transition DRAFT → GENERATED via transitionEntry(entry, 'generateEntry')
- Set editWindowExpiresAt via computeEditWindowExpiry
- Set pdfGenerated, pdfGeneratedAt, committedAtISO
- Compute streakEligible based on endDate
- Write store, update index, append WAL

**If missing: `finalizeEntry(email, category, entryId)`**
- Verify entry is GENERATED and has valid PDF
- Mark as finalized (set permanentlyLocked if second finalization)
- Write store, update index, append WAL

**If missing: `requestEdit(email, category, entryId, message?)`**
- Verify entry is finalized GENERATED
- Check request count limit (max 3/month)
- Transition via transitionEntry(entry, 'requestEdit')
- Handle streak permanent removal if entry is a Win
- Write store, update index, append WAL

**If missing: `requestDelete(email, category, entryId)`**
- Verify entry is finalized GENERATED
- Check request count limit
- Transition via transitionEntry(entry, 'requestDelete')
- Cancel any pending edit request
- Handle streak permanent removal if entry is a Win
- Write store, update index, append WAL

**If missing: `deleteEntry(email, category, entryId)`**
- Verify entry is deletable (DRAFT, editable GENERATED, or EDIT_GRANTED)
- Remove from store.byId and store.order
- Write store, update index, append WAL

### Every function must:
- Use `withUserLock` for concurrency safety
- Call `transitionEntry` from workflow.ts for status changes
- Set `updatedAt` on every mutation
- Write via `atomicWriteTextFile` (through dataStore)
- Update user index
- Append to WAL
- Return the complete entry (not a partial)

### Re-export from lifecycle.ts
For every new function added to engine.ts, add a re-export in `lib/entries/lifecycle.ts`.
