## The Fix

### Replace every direct file operation in categoryRouteHandler.ts

For each operation, replace the direct file code with an engine call:

**GET (list entries):**
```typescript
// BEFORE: reads JSON file directly
const store = await readCategoryStore(email, category);
const entries = Object.values(store.byId);

// AFTER: use engine or keep direct read (reads are acceptable)
// If engine has listEntries, use it. Otherwise direct read + toApiResponse is fine.
const entries = await listEntries(email, category);
return NextResponse.json(entriesToApiResponse(entries, category));
```

**POST (create):**
```typescript
// BEFORE: manually builds entry, writes to file
const entry = { id: uuid(), ...body, confirmationStatus: 'DRAFT', ... };
store.byId[entry.id] = entry;
await atomicWrite(path, store);

// AFTER:
const entry = await createEntry(email, category, body);
return NextResponse.json(entryToApiResponse(entry, category));
```

**PATCH (update/generate/finalise/request):**
```typescript
// BEFORE: reads file, switches on action, writes file
// This is the most complex handler with multiple branches

// AFTER: each action delegates to its engine function
if (action === 'save') {
  const updated = await updateEntry(email, category, entryId, fields);
  return NextResponse.json(entryToApiResponse(updated, category));
}
if (action === 'generate') {
  const generated = await generateEntry(email, category, entryId);
  return NextResponse.json(entryToApiResponse(generated, category));
}
if (action === 'finalise') {
  const finalized = await finalizeEntry(email, category, entryId);
  return NextResponse.json(entryToApiResponse(finalized, category));
}
if (action === 'requestEdit') {
  const requested = await requestEdit(email, category, entryId, body.message);
  return NextResponse.json(entryToApiResponse(requested, category));
}
if (action === 'requestDelete') {
  const requested = await requestDelete(email, category, entryId);
  return NextResponse.json(entryToApiResponse(requested, category));
}
```

**DELETE:**
```typescript
// BEFORE: reads file, removes entry, writes file

// AFTER:
await deleteEntry(email, category, entryId);
return NextResponse.json({ success: true });
```

### Remove all file system imports from the shared handler

After rewiring, the shared handler should NOT import:
- fs/promises
- lib/dataStore.ts (except maybe for read-only list)
- lib/data/fileAtomic.ts
- lib/data/locks.ts

It should ONLY import from:
- lib/entries/lifecycle.ts (engine operations)
- lib/entries/toApiResponse.ts (response formatting)
- lib/auth.ts (authentication)
- lib/security/ (rate limiting, payload validation)

### CRITICAL: The entryToApiResponse call happens in the handler, NOT in the engine
Engine returns raw entries. The handler wraps them with computed fields. This keeps the engine pure (no HTTP concerns) and the handler thin (no business logic).
