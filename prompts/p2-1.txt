## Context
The app has 5 nearly identical API route files (fdp-attended, fdp-conducted, guest-lectures, case-studies, workshops), each 690-1075 lines. They share 90%+ of their logic — only the category key, field names, and validation schema differ.

This prompt creates a shared handler module that all 5 routes will delegate to. We start with GET (list entries) and POST (create entry).

## The Implementation

Create `lib/api/categoryRouteHandler.ts`:

```typescript
/**
 * Shared route handler for all 5 category API routes.
 * 
 * Each category route becomes a thin wrapper:
 *   export const GET = (req) => handleCategoryGet(req, 'fdp-attended');
 *   export const POST = (req) => handleCategoryPost(req, 'fdp-attended');
 * 
 * This module owns:
 * - Auth checks
 * - Category validation
 * - Schema-based field validation
 * - Delegation to engine.ts for persistence
 * - Response formatting via entryToApiResponse
 * 
 * This module does NOT own:
 * - Business rules (those live in workflow.ts)
 * - Persistence internals (those live in engine.ts)
 * - PDF generation (that lives in pdfService.ts)
 */
```

### handleCategoryGet(req, categoryKey)
Extract the GET handler logic from fdp-attended/route.ts:
1. Authenticate via `getServerSession(authOptions)` — return 401 if no session
2. Validate categoryKey against the category registry — return 400 if invalid
3. Read the user's category store file (same pattern as current route)
4. Convert each entry via `entryToApiResponse(entry, categoryKey)`
5. Return the list as JSON

### handleCategoryPost(req, categoryKey)
Extract the POST handler logic from fdp-attended/route.ts:
1. Authenticate — return 401 if no session
2. Rate limit check — use the `entryMutations` preset
3. Validate payload size via `assertEntryMutationInput`
4. Validate categoryKey
5. Get the category schema, validate the body against it
6. Create entry via engine.ts `createEntry` (NOT direct JSON write)
   - If engine.ts doesn't have a suitable createEntry, create the entry object manually but still use the atomic write pattern from the current route
7. Set initial lifecycle fields: id, category, ownerEmail, createdAt, updatedAt, confirmationStatus: 'DRAFT'
8. Save to the category store file
9. Return `entryToApiResponse(savedEntry, categoryKey)`
