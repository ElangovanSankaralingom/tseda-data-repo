## Context
Three components have hardcoded per-category metadata:

1. **CommandPalette.tsx** (lines 37-41) — hardcoded quick actions per category
2. **CategoryCard.tsx** (lines 17-45) — hardcoded icon, color, description per category
3. **EditorProgressHeader.tsx** (lines 7-9) — hardcoded color themes per category

When you add a new category, all three break silently (no error, just missing UI).

## Read First
1. components/search/CommandPalette.tsx (hardcoded quick actions)
2. components/dashboard/CategoryCard.tsx (hardcoded metadata)
3. components/data-entry/EditorProgressHeader.tsx (hardcoded colors)
4. data/categoryRegistry.ts (where UI metadata should live)

## The Fix

### Step 1: Add UI metadata to category registry

In `data/categoryRegistry.ts`, extend each category config:
```typescript
{
  key: 'fdp-attended',
  label: 'FDP Attended',
  shortLabel: 'FDP (A)',
  description: 'Faculty development programs you attended',
  icon: 'GraduationCap', // lucide-react icon name
  color: {
    bar: 'from-blue-400 to-blue-600',
    bg: 'bg-blue-100',
    text: 'text-blue-600',
    accent: 'blue',
  },
}
```

### Step 2: CommandPalette reads from registry
```typescript
// BEFORE: hardcoded array

  { id: "qa-fdp-a", title: "New FDP Attended Entry", path: "/data-entry/fdp-attended/new" },

// AFTER: derived from registry
import { CATEGORY_LIST } from '@/data/categoryRegistry';

```
