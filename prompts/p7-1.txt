### Test categories:

**Create:**
- Creates entry with correct initial fields
- Sets confirmationStatus to DRAFT
- Generates unique ID
- Sets createdAt and updatedAt

**Update:**
- Updates allowed fields
- Rejects update on non-editable entry
- Sets updatedAt
- Does not overwrite lifecycle fields

**Generate (DRAFT → GENERATED):**
- Sets confirmationStatus to GENERATED
- Sets editWindowExpiresAt (3 days for non-streak)
- Sets pdfGenerated, pdfGeneratedAt, committedAtISO
- Sets streakEligible if endDate is future
- Does NOT set streakEligible if endDate is past

**Finalise:**
- Succeeds for GENERATED entry with valid PDF
- Sets permanentlyLocked on second finalization
- Rejects if no PDF
- Rejects if PDF is stale

**Request edit:**
- Transitions GENERATED (finalized) → EDIT_REQUESTED
- Increments requestCount
- Sets streakPermanentlyRemoved if entry is a Win
- Rejects if not finalized
- Rejects if at request limit (3/month)

**Request delete:**
- Transitions GENERATED (finalized) → DELETE_REQUESTED
- Cancels pending edit request
- Rejects if not finalized

**Delete:**
- Deletes DRAFT entries
- Deletes editable GENERATED entries
- Rejects delete on finalized entries

**Archive / Restore:**
- Archives entry, sets archivedAt
- Restore resets to GENERATED, clears PDF state
