## PROJECT OVERVIEW

**TSEDA** = Gamified faculty data collection app for **TCE (Thiagarajar College of Engineering)**, Madurai.

Faculty members log professional development activities (FDPs attended, FDPs conducted, guest lectures, case studies, workshops). The app gamifies data entry with a streak system to encourage timely submissions.

**Current state: Production-ready. Audit score 8.8/10 across 18 categories.**

## ARCHITECTURE (FINAL STATE)

### Unified Route Handler
- `lib/api/categoryRouteHandler.ts` — single shared handler (auth, CSRF, validation, rate limiting, pagination)
- 5 routes at `app/api/me/*/route.ts` — 25 lines each (thin wrappers)
- `lib/api/withErrorHandler.ts` — global error wrapper with timing and request ID
- `lib/api/response.ts` — standard response envelope
- `lib/api/asyncContext.ts` — request ID via AsyncLocalStorage

### Engine (Single Entry Point)
ALL mutations flow through `lib/entries/internal/engine.ts` (barrel):
- `engineRead.ts` — list, get
- `engineWrite.ts` — create, update, delete
- `engineCommit.ts` — commit, finalise
- `engineTransitions.ts` — status transitions
- `engineAdmin.ts` — admin operations
- `engineRequests.ts` — edit/delete requests
- `engineValidation.ts` — input validation
- `engineHelpers.ts` — shared utilities
- `engineIndex.ts` — index and WAL
- `engineMutationRunner.ts` — mutation orchestration

Public API: `lib/entries/lifecycle.ts`

### Type System
- `lib/types/entry.ts` — CanonicalEntry, EntryApiResponse, EntryLifecycleFields, EntryStatus, FieldStage
- `lib/types/admin.ts`, `lib/types/dashboard.ts` — domain types
- `lib/entries/toApiResponse.ts` — entry → API response converter
- `lib/api/contracts.ts` — Zod request/response schemas (planned)
- **0 `any` types in entire codebase**
- Strict tsconfig enabled

### Schema System
- `data/schemas/*.ts` — field definitions with `stage: 1` (data) / `stage: 2` (uploads)
- `data/categoryRegistry.ts` — category config with icon, color, description, label
- `lib/validation/schemaValidator.ts` — schema-driven validation (auto-enforces required fields)
- `lib/pdfSnapshot.ts` — schema-driven PDF hash (reads stage from schemas)

### Security
- `lib/security/csrf.ts` — CSRF protection on all mutations
- `lib/security/fileValidation.ts` — MIME type, size, content validation
- `lib/security/sanitize.ts` — HTML stripping, length truncation, null byte removal
- `lib/security/rateLimit.ts` — rate limits on ALL endpoints (GET, mutations, uploads, admin)
- Security headers: X-Frame-Options, X-Content-Type-Options, CSP, Referrer-Policy, HSTS
- Session: 8-hour maxAge, httpOnly, sameSite cookies

### Error & Resilience
- `lib/errors.ts` — AppError hierarchy (ValidationError, NotFoundError, ForbiddenError, RateLimitError, ConflictError)
- `lib/api/withErrorHandler.ts` — global API error wrapper
- `instrumentation.ts` — unhandledRejection + uncaughtException handlers
- `lib/utils/timeout.ts` — timeout wrapper for all async ops
- `lib/utils/fireAndForget.ts` — safe fire-and-forget with error logging
- `lib/server/shutdown.ts` — graceful shutdown on SIGTERM/SIGINT

### Observability
- `lib/api/asyncContext.ts` — request ID via AsyncLocalStorage
- `lib/logger.ts` — structured JSON logger with levels, auto-includes requestId
- `lib/observability/errorReporter.ts` — error reporting (Sentry-ready)
- `lib/observability/scrubber.ts` — PII scrubbing for logs
- 0 console.log in codebase
- Response timing on all API calls

### Data Storage
- `lib/data/dataLayer.ts` — abstract DataLayer interface
- `lib/data/jsonDataLayer.ts` — current JSON file implementation
- `.data/` gitignored, not in repo
- `lib/config/appConfig.ts` — all config values centralized

### Background Jobs
- `app/api/cron/nightly/route.ts` — auto-archive, edit grant expiry, timer warnings, WAL compaction
- `lib/notifications/create.ts` — notification creation

### CI/CD
- `.github/workflows/ci.yml` — lint, typecheck, build, test, audit, docker
- `.github/workflows/release.yml` — auto-release on tags
- `.github/dependabot.yml` — weekly dependency updates
- `Dockerfile` + `docker-compose.yml` + `.dockerignore`
- `scripts/deploy.sh` + `scripts/rollback.sh`

### Frontend
- `components/ui/ErrorBoundary.tsx` — error boundaries
- `@next/bundle-analyzer` configured
- `@axe-core/react` for accessibility
- `components/shell/` — TopNav, Sidebar, NotificationBell, UserMenu
- `components/account/` — split account sections
